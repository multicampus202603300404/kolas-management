# [PRD] KOLAS 인증 기관용 통합 시험 관리 플랫폼 (K-LIMS)

- **제품명:** K-LIMS (KOLAS Laboratory Information Management System)
- **버전:** v1.0
- **목적:** 시험 의뢰부터 성적서 발행, 수수료 정산 및 전 과정의 전자결재화를 통한 KOLAS 17025 준수 및 운영 효율화
- **대상:** KOLAS 인증 시험소 직원(시험원, 기술/품질책임자), 시험 의뢰 고객

---

## 1. 핵심 프로세스 흐름 (End-to-End)

```
고객          온라인 의뢰서 작성 및 견적 확인
   ↓
접수 담당     시료 확인 및 계약 검토  →  [결재 1: 계약 승인]
   ↓
시험원        시료 배정 및 시험 수행 (Raw Data 입력, 불확도 계산)
   ↓
검토자        시험 기록서 검토 및 승인  →  [결재 2: 데이터 검토]
   ↓
승인자        최종 성적서 승인 및 발행  →  [결재 3: 성적서 승인]
   ↓
정산          수수료 청구 및 세금계산서 발행
```

---

## 2. 주요 기능 요구사항

### 2.1 고객 포털 및 의뢰 관리 (CRM)

- **온라인 의뢰:** 고객이 시험 항목, 시료 정보, 희망 완료일 직접 입력
- **견적 자동 산출:** 시험 항목별 단가 DB를 기반으로 견적서 PDF 생성 및 전송
- **진행 상태 조회:** 접수 → 시험 중 → 검토 중 → 발행 완료 상태 실시간 노출

### 2.2 접수 및 시료 관리

- **계약 검토:** 시험 가능 여부(장비, 인력, 환경)를 체크하는 '계약 검토서' 작성
- **시료 ID 부여:** 입고 시 고유 식별 번호(QR/바코드) 자동 생성 및 라벨 출력
- **환경 기록:** 시험실 내 온·습도 센서 연동을 통한 실시간 환경 데이터 로깅

### 2.3 시험 수행 및 데이터 무결성 (LIMS Core)

- **워크시트(Worksheet):** 규격별 표준 계산식 및 측정불확도 산출 로직 내장
- **장비 연동:** 계측기(RS232/USB/LAN) 데이터 자동 수집으로 전사 오류 원천 차단
- **Audit Trail:** 모든 수치 수정 이력에 대해 '수정 전/후, 수정자, 사유'를 로그로 보관

### 2.4 통합 전자결재 시스템 (e-Approval)

- **결재 라인 커스텀:** [기안 - 검토 - 승인] 기본 라인 외에 전결 규정에 따른 유연한 설정
- **디지털 서명:** 전자서명법을 준수하는 암호화된 서명 적용
- **반려 프로세스:** 반려 시 사유 입력을 강제하며, 기안자에게 즉시 알림 전송

### 2.5 성적서 및 정산 관리

- **성적서 자동 생성:** 승인된 Raw Data를 성적서 템플릿에 자동 매핑 (PDF 변환)
- **위변조 방지:** 성적서 진위 확인 QR코드 및 복사 방지 워터마크 적용
- **수수료 관리:** 입금 확인 여부에 따른 성적서 출력 제어 로직 구현

## 3. UI 화면 설계

| 화면 ID | 화면명 | 주요 기능 |
|---------|--------|-----------|
| **UI-01** | **종합 대시보드** | 전체 진행 현황, 미결재 문서 건수, 장비 교정 예정 알림 |
| **UI-02** | **시험 관리 시트** | 시료별 결과 값 입력, 기준치 대비 합격/부적합 자동 판정 |
| **UI-03** | **통합 결재함** | 기안/진행/반려/완료 문서함, 결재 의견 작성란 포함 |
| **UI-04** | **장비/소모품 관리** | 장비 이력 카드, 교정 주기, 표준물질 재고 현황 관리 |

## 4. 기술 요구사항 및 보안

- **데이터베이스:** 수정 불가능한 로그 기록을 위한 Audit Log Table 별도 운영
- **보안:** 데이터 전송 구간 암호화(SSL), 개인정보(고객사 담당자 정보) 암호화 저장
- **접근 제어:** RBAC(Role-Based Access Control) 기반의 권한 관리 (시험원 vs 책임자)
- **백업:** 일 단위 전체 백업 및 실시간 트랜잭션 로그 백업

---

## 5. KOLAS 준수 (Compliance)

- **유효성 검증(Validation):** 시스템의 계산 로직(불확도 등)이 정확한지 검증한 보고서 기능 제공
- **이력 추적성:** 성적서 번호 하나로 [의뢰서-장비-시험원-결재이력-Raw Data]가 모두 연결되어야 함
- **성적서 개정 관리:** 발행 후 수정 시 사유 기록 및 기존 성적서 무효화 처리 프로세스 강제

---

## 6. 단계별 구축 로드맵

| 단계 | 내용 |
|------|------|
| **Phase 1 (기초)** | 의뢰/접수/시료 관리 및 기본 워크시트 개발 |
| **Phase 2 (핵심)** | 전자결재 시스템 연동 및 성적서 자동 발행 엔진 구축 |
| **Phase 3 (고도화)** | 계측 장비 인터페이스(IoT) 연동 및 수수료 정산 모듈 통합 |
